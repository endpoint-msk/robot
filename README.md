# endpoint-robot

Телеграм-бот хакерспейса на TypeScript + [mtcute](https://mtcute.dev): сборы донатов, присутствие резидентов, статус 3D-принтера и миниапп хостинга гостей. Стейт — один JSON-файл.

Работает только в чатах из `ALLOWED_CHATS`, в остальных молчит. Резиденты — админы этих чатов.

## Сборы

Один сбор на период, по умолчанию — месяц UTC.

- `/goals` — показать текущий сбор
- `/donate <сумма> <ник>` — добавить донат; без ника — анонимный
- `/remove <номер>` — удалить донаты участника по месту в лидерборде
- `/remove <ник> [сумма]` — удалить один донат
- `/setgoal <сумма>` — цель, `0` снимает
- `/settitle <тема>` — тема сбора, по умолчанию «аренду»
- `/setdesc <текст>` — описание под сбором (реквизиты, многострочное); без текста убирает
- `/setresetday <1–29>` — день сброса, по умолчанию 1
- `/goalsmute` — вкл/выкл ежедневный постинг в этот чат

Лидерборд группируется по нику и сортируется по сумме, топ-3 — 🥇🥈🥉, при достижении цели — «✅ Сбор закрыт». Сообщение перерисовывается на месте при правках, при смене периода переключается на новый сбор. Каждый день в 00:00 и 12:00 МСК постится свежее сообщение. Всё, кроме `/goals` и `/help`, — только для админов.

## Присутствие

- `/start` в личке — меню «Отметиться с ником / без ника»
- `/inside` — показать разовый список тех, кто в спейсе; доступна всем
- `/bindmac <MAC>`, `/unbindmac`, `/maclist`, `/settings` — авто-отметки по MAC

Кто сейчас внутри показывается на закреплённой доске «кто сегодня в спейсе» (блок «Сейчас в спейсе»); чек-ин/чек-аут её обновляют. Отдельной авто-рассылки списка больше нет — выключить доску в чате можно через `/boardmute`. Каждые 3 часа бот пингует в личку: «Я внутри» продлевает, 15 минут без ответа — отметка снимается.

Авто-отметки по MAC: бот опрашивает роутер Keenetic и отмечает резидента, пока устройство в сети, снимает через 10 минут отсутствия. Такие отметки не пингуются. На устройстве нужно выключить рандомизацию MAC для Wi-Fi спейса.

## 3D-принтер

`/printer` — статус из Moonraker (Klipper); в личке и в allowlist-чатах. При активной печати показывает прогресс, превью (миниатюра gcode / снимок вебки, переключаются кнопками) и кнопку «Уведомить по окончании» — подписчик получит сообщение в личку, когда печать завершится.

## Хостинг гостей (миниапп)

Гость оставляет заявку на визит, резидент её одобряет.

- Гость: день из ближайших 7, время и цель. Видит только свои визиты; когда заявку одобрят — бот напишет, кто хостит.
- Резидент: обзор на 7 дней со всеми заявками, детали дня, кнопка «Захостить». Одобривший может снять своё одобрение. Архив прошедших недель — только просмотр.
- Уведомления о новых заявках приходят в личку, по умолчанию только о заявках на сегодня; в настройках — все либо выкл.
- Настройки авто-отметки по MAC продублированы в миниаппе, стейт общий с `/bindmac` и `/settings`.
- Тема системная, в настройках можно закрепить светлую или тёмную. Гостю в настройках доступна только тема.
- В подтверждённом визите есть «Добавить в календарь» — `.ics` по подписанной ссылке.
- Для `DEV_USER_IDS` есть дев-меню: перспектива «резидент ↔ гость», фейковые заявки, правка и удаление любых заявок.

Кнопки входа: рядом с полем ввода в личке, под списком `/inside` и в `/start`.

Включается `WEBAPP_URL` — публичный HTTPS-адрес, под которым отдаётся собранный миниапп из `webapp/dist`. Бот поднимает HTTP-сервер на `WEBAPP_HOST:WEBAPP_PORT`, наружу его выставляет реверс-прокси с TLS. Чтобы кнопки из групп открывали миниапп, включи Main Mini App в BotFather (Bot Settings → Configure Mini App) с тем же URL. Авторизация запросов — подпись `initData`.

## Запуск

```bash
cp .env.example .env          # API_ID, API_HASH, BOT_TOKEN, ALLOWED_CHATS
npm install
npm --prefix webapp install   # зависимости миниаппа (React + Vite)
npm --prefix webapp run build  # сборка миниаппа в webapp/dist (если задан WEBAPP_URL)
npm start                     # npm run dev — watch-режим
```

Миниапп хостинга — отдельное React + Vite + TypeScript приложение в `webapp/`; сервер бота раздаёт готовую сборку из `webapp/dist`. Для разработки фронта — `npm --prefix webapp run dev` (Vite dev-сервер с прокси `/api`, `/avatar.jpg`, `/visit.ics` на бэкенд). В Docker сборка миниаппа — шаг в `Dockerfile`.

Остальные переменные — в `.env.example`, каждая выключает свою подсистему, если не задана: `PRINTER_URL`/`PRINTER_AUTH`, `KEENETIC_URL`/`KEENETIC_LOGIN`/`KEENETIC_PASSWORD`, `WEBAPP_URL`/`WEBAPP_HOST`/`WEBAPP_PORT`/`HOSTING_TZ_OFFSET_MINUTES`, `DEV_USER_IDS`, `DATA_FILE`, форвардинг и live-чат.

Стейт — в `DATA_FILE` (по умолчанию `./data.json`), сессия mtcute — в `bot.session`. Тестов и линтера нет, проверка — `npm run typecheck`.

## Структура

```
src/index.ts       — точка входа: env, клиент, диспетчер, хендлеры, шедулеры
src/handlers.ts    — команды сборов, пагинация, проверки allowlist/админа
src/fundraiser.ts  — модель сбора: лидерборд, прогресс-бар, периоды, парсинг
src/presence.ts    — присутствие: меню, пинги, авто-постинг списка
src/residents.ts   — ResidentDirectory: кто резидент и кто админ
src/keenetic.ts    — клиент RCI-API Keenetic для авто-отметок по MAC
src/printer.ts     — /printer, Moonraker, поллер окончания печати
src/hosting.ts     — модель хостинга: заявки, дни/недели, архив, .ics
src/webapp.ts      — HTTP-сервер миниаппа: статика webapp/dist + JSON API
src/scheduler.ts   — смена периода + ежедневный постинг сбора
src/storage.ts     — JSON-хранилище с атомарной записью
src/types.ts       — типы стейта
webapp/            — фронт миниаппа (React + Vite + TypeScript, iOS, светлая и тёмная темы)
```
